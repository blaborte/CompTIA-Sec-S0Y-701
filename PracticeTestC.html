⚡ PERFORMANCE-BASED QUESTIONS (C1–C5)

These interactive PBQ questions require matching, drag-and-drop, or configuration tasks on the real exam. Answers are summarized below for study purposes.
C1 — FIREWALL RULES: ALLOWED OR BLOCKED?
  • • Use a secure terminal to connect to 10.1.10.88 → ALLOWED (SSH = TCP/22, Rule 1)
  • • Share the desktop on server 10.1.10.120 → BLOCKED (RDP = TCP/3389, not allowed to .120)
  • • Perform a DNS query from 10.1.10.88 to 9.9.9.9 → ALLOWED (DNS = UDP/53, Rule 5)
  • • View web pages on 10.1.10.120 → ALLOWED (HTTP/HTTPS = TCP/80,443, Rules 2 & 3)
  • • Authenticate to an LDAP server at 10.1.10.61 → BLOCKED (LDAP = TCP/389, no matching rule)
  • • Synchronize the clock on a server at 10.1.10.17 → ALLOWED (NTP = UDP/123, Rule 6)
C2 — MATCH DEVICE TO DESCRIPTION
  • • Block SQL injection over an Internet connection → IPS
  • • Intercept all browser requests and cache the results → Proxy server
  • • Forward packets between separate VLANs → Router
  • • Configure a group of redundant web servers → Load balancer
  • • Evaluate the input to a browser-based application → WAF
C3 — MATCH ATTACK TYPE TO CHARACTERISTIC
  • • A website stops responding to normal requests → DDoS
  • • Data is captured and retransmitted to a server → Replay
  • • Malware designed to remain hidden on a system → Rootkit
  • • A list of passwords attempted with a known username → Brute force
  • • Email link redirects user to a site requesting login credentials → Phishing
  • • Permissions circumvented by adding code as application input → Injection
C4 — MATCH CRYPTOGRAPHY TECHNOLOGY
  • • Create a stronger key using multiple processes → Key stretching
  • • Data is hidden within another media type → Steganography
  • • Different inputs create the same hash → Collision
  • • Sensitive data is hidden from view → Masking
  • • A different key is used for decryption than encryption → Asymmetric
  • • Information is added to make a unique hash → Salting
C5 — MATCH SECURITY TECHNOLOGY TO SCENARIO
  • • Field engineer uses corporate laptop at coffee shops/hotels → VPN
  • • Developers run tests before deploying an application → Sandboxing
  • • Prevent employees from visiting known-malicious websites → NGFW
  • • Directly access cloud-based services from all corporate locations → SD-WAN
  • • Users connecting to network should use corporate credentials → 802.1X
