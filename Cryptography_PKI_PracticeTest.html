This quiz covers Section 9: Cryptography & PKI — PKI hierarchy (CA, RA, Root CA, Sub-CA), certificate revocation (CRL, OCSP, Stapling), certificate formats (PEM/DER/PFX/P7B/CSR), symmetric vs. asymmetric algorithms (AES/RSA/ECC), cipher modes (ECB/CBC/GCM), hashing (SHA-256/MD5), key stretching (bcrypt/PBKDF2), Diffie-Hellman key exchange, Perfect Forward Secrecy, and key management (HSM/Key Escrow/DRA).

Tip: ECB vs CBC, CRL vs OCSP vs Stapling, and symmetric vs asymmetric use cases are classic exam traps.

C9-3 — CERTIFICATE FORMATS: IDENTIFY THE CORRECT FORMAT
For each scenario, select the appropriate certificate format or file type.
#SCENARIOFORMAT
1A Linux web admin needs to configure an Apache web server with an SSL certificate. The format must be Base64-encoded and human-readable. Common extensions are .pem, .crt, or .cer.
2A Windows administrator needs to export a certificate WITH its private key and the entire certificate chain into a single encrypted file for backup and transfer. The file will have a .pfx extension.
3A developer needs to submit a certificate request to a CA for a new server certificate. The file contains the public key and identity information but no certificate yet.
4A Java Tomcat administrator needs to import a certificate chain. The format is Base64-encoded, can store the chain but NOT the private key, and uses .p7b or .p7c extensions.
5A Java application requires a certificate in binary (non-human-readable) format. The admin provides a file with a .der extension that cannot be opened as text.
