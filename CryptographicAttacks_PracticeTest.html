CA10-3 — ATTACK SCENARIOS: IDENTIFY THE CORRECT ATTACK
For each real-world scenario, select the cryptographic attack being described.
#SCENARIOATTACK TYPE
1A researcher compromises a TLS session by positioning themselves between a client and server, silently decrypting, reading, and re-encrypting traffic in real time. Neither party is aware of the interception.
2An analyst uses a power-monitoring device next to an HSM and notices that the device draws slightly different electrical current during different stages of an RSA private key operation. They use the power traces to infer bits of the private key.
3A cryptanalyst obtains a database of intercepted messages from WWII Enigma communications along with some known plaintext messages (weather reports with predictable phrasing). They use these pairs to deduce the Enigma key settings.
4A security researcher submits two different PDF documents to an MD5 hash function and discovers they both produce the identical hash value — allowing a malicious PDF to be swapped for a legitimate one without detection.
5An attacker compromised a 2DES implementation by encrypting all possible keys from one end and decrypting with all possible keys from the other end, then matching results in the middle — reducing the effective key search space from 2^112 to approximately 2^57.
